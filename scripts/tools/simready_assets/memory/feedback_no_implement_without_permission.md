name: Do Not Implement Without Permission
description: Always discuss approach before coding. Don't implement options or create new systems without user approval.
type: feedback
---

When the user says "let's discuss" — discuss, don't code.

**Why:** User asked to discuss Track B (physics reasoning for drive parameters). I jumped to implementing Option C (lookup table with guessed physics values) without asking. The values were untested guesses that would need sim validation. User rightfully asked why I created it without permission.

**How to apply:** When multiple implementation options exist, present them clearly and wait for the user to choose. Don't pick an option and implement it during a discussion. Only code after explicit approval of the approach.
